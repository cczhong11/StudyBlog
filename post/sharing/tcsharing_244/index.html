新闻思考

这周我看了一个新出的纪录片，是讲 Python 的历史的。作为一个喜欢历史也喜欢使用 Python 的人，我对这部片子非常着迷，它让我看见了一门语言成长过程中的起起伏伏。

最让我印象深刻的第一个故事，Guido van Rossum 在 CWI（荷兰国家数学与计算机科学研究所）工作时，团队中的 ABC 语言是他的早期项目之一。然而 ABC 过于“体贴入微”却过于僵化——难以扩展、难以部署，让人望而却步。当时他手上有个 side project，就是后来成为 Python 的雏形。虽然老板并不看好，但他坚持开发，并把它放到网上公开，结果瞬间获得了大家的喜爱。他借鉴了 ABC 语言的优点，却避免了它传播受限的缺点，这让我学到一个道理：如果你真的希望一个东西被更多人使用，选择合适的传播方式比什么都重要。

第二个故事是关于 Anaconda 的诞生。早期科研人员使用 Python 做科学研究时需要自己安装依赖，过程繁琐，严重影响了传播。直到 Anaconda 出现，它把 NumPy、SciPy 等常用库打包好，让科研人员可以开箱即用，逐渐取代了 Matlab，成为科学研究的重要工具。这让我体会到，好的工具不仅在于功能强大，更要让人容易上手。

第三个故事发生在一次 PyCon 大会上，Guido 提到核心开发团队里没有女性，这句话触动了 Mariatta。她虽然紧张，却还是鼓起勇气给 Guido 写了一封邮件，没想到 Guido 回复了她，还一步步指引她如何成为核心开发者。最终她成为第一位女性核心成员。让我学到的就是：有的时候，你不需要想得太远，只要勇敢去做就好。因为未来会发生什么，其实谁都不知道。就像 Mariatta，如果她因为紧张而退缩，就不会有后来那一切。勇气不是因为确定会成功，而是因为你愿意去尝试。

第四个故事讲的是 Python 2 向 Python 3 的迁移。Python 3.0 在 2008 年发布，带来了很多“破坏性更改”（breaking changes），最著名的就是 字符串与 Unicode 的重构：在 Python 3 中，str 默认是 Unicode，而 Python 2 则默认是字节串。这对于开发者来说意味着——所有字符串处理代码都要改，成本极高。因为改动大、迁移成本高，而一开始又缺乏足够吸引人的新特性，很多人宁愿留在 Python 2，不愿意动。但随着 Python 3.4 带来了如 asyncio、以及后来 f-string 等“眼前一亮”的功能，才让开发者们觉得迁移值得，最终才完成了这场漫长的过渡。这让我学到，在软件开发中，迁移不靠宣传，而靠真实的价值。

第五个故事是关于海象运算符 PEP 572。这个小小的 := 运算符引起了巨大的争论，Guido 最终拍板接受，却因此背负了沉重的压力，最后辞去了 BDFL 的职位。在 Python 社区里，Guido van Rossum 被称为 BDFL（Benevolent Dictator For Life，终身仁慈独裁者）。这个职位意味着，在 Python 语言的发展过程中，每一个 PEP（Python Enhancement Proposal，增强提案）最终是否能被采纳，都要由 Guido 拍板。他并不是每个 PEP 都亲自写代码，但他是那个“最后的审批人”。就像一位“最高法官”，无论社区里讨论得多么激烈，最终都要由他来点头或者摇头。在 PEP 572（海象运算符） 的争论里，这个机制表现得淋漓尽致。社区里意见两极化，大家争得不可开交。但因为 Guido 是 BDFL，所以他必须给出最终决定。虽然他最后批准了这个提案，但第二天就引来更猛烈的争论和批评。压力积累到顶点，他也意识到：一个人背负所有的决定权力，同时也要承受所有的批评和责任。所以在PEP572之后，他辞去了 BDFL 职位。之后 Python 改用 Steering Council（指导委员会） 的治理模式，变成由一个小团队来共同拍板，避免把所有权力压在一个人肩上。自那以后，Python 由委员会来共同决策。这个故事让我明白，巨大的权力背后往往是巨大的责任，独裁并不意味着成功，有时候集体的智慧才是更长远的保障。

整部纪录片让我在 Python 的历史中看到了坚持、勇气、易用性、价值驱动和社区协作这些关键词，也让我在学习和使用这门语言时更有感情。

【1】华为云突发重大重组，盘古大模型等数十个部门被裁撤，波及人数或达上千 据“拟合论见”披露，华为云已启动一轮声势浩大的组织调整，波及人数或达上千。就在前一日，公司下发内部正式通知，宣布对云 BU 的产品、公有云服务、研发等多条关键业务线进行撤并，数十个部门将随之被裁撤出局。最引人关注的是，云 EI 产品线下的盘古大模型相关部门被明确撤销。此前，盘古大模型曾因“套壳”质疑和前员工爆料管理混乱而陷入舆论漩涡。如今这份带有强烈收缩意味的正式决定，被普遍解读为华为对盘古项目态度的重大转折点。目前，华为内部的活水已经启动，众多华为其他业务线的HR，以及其他大厂正在小红书上在线捞人。
